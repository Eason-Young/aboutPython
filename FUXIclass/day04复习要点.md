01-多任务编程

```
同步与异步、阻塞与非阻塞
	程序运行中的同步与异步刚好与生活中的同步与异步概念相反
	
	阻塞：在程序运行的过程中，调用一个功能后，必须等这个功能执行完成之后再继续执行程序中后续的功能代码
	非阻塞：在程序运行的过程中，调用一个功能后，不必等这个功能执行完成，就可以马上执行程序后后续的功能代码

Python中实现多任务的三种方式：多进程、多线程、协程
	多进程：使用 multiprocessing 模块
	多线程：使用 threading 模块
	协程：使用yield【配合next或send手动调用】、greenlet【配合switch手动调用】、gevnet【使用gevent时需要打补丁，自动调用】

多进程，多线程与协程的区别：
	多进程与多线程时操作系统或者说时CPU自动切换调用
	协程实际上是通过代码逻辑做切换调用

多进程、多线程、协程的区别与优劣势：
	https://blog.csdn.net/qq_31603575/article/details/80223950

使用场景：
	多进程：CPU【计算资源】密集型的操作
	多线程：IO【阻塞】密集型的操作
	协程：IO【阻塞】密集型的操作

并发与并行：
	并发：同一时间段内执行多个任务【任务数量大于CPU核数】
	并行：同一时间点上执行多个任务【任务数量小于或等于CPU核数】

GIL全局解释器锁：历史遗留，CPython解释器的问题，无论你启多少个线程，你有多少个cpu, CPython在执行的时候会淡定的在同一时刻只允许一个线程运行。

如何避开GIL：
	推荐：多进程+协程
	不推荐：更换其他语言开发的Python解释器
	嵌入其他语言的代码，
	例如调用C/C++：https://blog.csdn.net/u012247418/article/details/80170690
	调用Golang：https://blog.csdn.net/gtd138/article/details/79801235

互斥锁与死锁：
	互斥锁：将资源锁定，未释放之前不运行其他进程或线程使用
	死锁：一般发生在有多个互斥锁时，即多个互斥锁相互钳制，都无法解锁

使用队列【queue】做本地进程间的通信（生产者消费者模式）：
当你要实现一个复杂的功能时，可以先写一个简单的案例测试你要使用到的模块
http://www.cnblogs.com/itogo/p/5635629.html
https://docs.python.org/3/library/asyncio-queue.html
当使用get方法获取到对象之后，使用完确认不在需要，可以调用task_done方法将该对象真正的从队列中删除

学习过程中发现：
	Python3 取消了cmp内建函数
	Python2和Python3关于queue模块有了较大的更新
```

05-正则表达式

```
常用的方法：
	match、search、findall、sub、split

匹配符：
    字符	功能
    .	匹配任意1个字符（除了\n）
    [ ]	匹配[ ]中列举的字符
    \d	匹配数字，即0-9
    \D	匹配非数字，即不是数字
    \s	匹配空白，即 空格，tab键
    \S	匹配非空白
    \w	匹配单词字符，即a-z、A-Z、0-9、_
    \W	匹配非单词字符
    
匹配多个字符：
    字符	功能
    *	匹配前一个字符出现0次或者无限次，即可有可无
    +	匹配前一个字符出现1次或者无限次，即至少有1次
    ?	匹配前一个字符出现1次或者0次，即要么有1次，要么没有
    {m}	匹配前一个字符出现m次
    {m,n}	匹配前一个字符出现从m到n次

匹配开头和结尾：
	字符	功能
    ^	匹配字符串开头
    $	匹配字符串结尾
  
取反： [^ ]
	
分组：
	字符			功能
    |			匹配左右任意一个表达式
    (ab)		将括号中字符作为一个分组
    \num		引用分组num的匹配规则
    (?P<name>)	分组起别名
    (?P=name)	引用别名为name分组的匹配规则
贪婪与非贪婪：默认使用贪婪模式，在控制匹配次数的符号后添加 ？问号表示使用非贪婪模式

原生字符串：r
```
